Markup

Rather than using the HTML provided in Bootstrap, Sherpa has custom code to better simplify and structure HTML. Paired with a data object such as the one above, one custom tag takes care of all of the work.

 

< div data-bind="component: { 
	name :'dropdown', 
	type :'button',
	data : simple_dropdown_data,
	options:{
		color : 'primary',
		size : 'large'
	}
}"></div>

Data behind dropdown

Puting data behind the dropdown makes this component dynamic and always easy to change without altering HTML code. Here is an example of a data object that could feed the dropdown samples above. This structure should be added to the prototype application assets/js/app.js.

simple_dropdown_data : {
    label : 'Sample button dropdown',
    items: [
        {
            id : "action_1",
            label : "Action 1",
            url : "#action"
        },
        {
            id : "action_2",
            label : "Action 2",
            url : "#action"
        },
        {
            id : "action_3",
            label : "Action 3",
            url : "#action",
            disabled: true
        },
        {
            divider :  true
        },
        {
            id : "action_4",
            label : "Action 4",
            url : "#action"
        }
    ]
}

Options

You may have noticed the data-bind tag above with various elements.

name:

This is always set "dropdown"

type:

Type provides options for the trigger mechanism. Whatever you see in the type correspinds to an HTML file name inside components/dropdown/whatever.html

The current types available for dropdown are:

  • button
  • split-button
  • link

options:

Options provide further customization of each type. Defualt options will be in place if options is not provided.The current options available for dropdowns are:

  • color:
    • primary
    • secondary
    • tertiary (default)
  • size:
    • large
    • default (default)
    • small
    • mini
  • orientation:
    • dropdown (default)
    • dropup

Sub menus on dropdowns

To add an extra level of dropdown menus the markup already takes care of it. All you have to do is have a different data set with nested elements.

 

sample_dropdown_1 : {
    label : 'Dropdown with submenus',
    items: [
        {
            id : "action_1",
            label : "Action 1",
            url : "#action"
        },
        {
            id : "action_2",
            label : "Action 2",
            url : "#action"
        },
        {
            id : "action_3",
            label : "Action 3",
            url : "#action",
            disabled: true
        },
        {
            divider :  true
        },
        {
            id : "action_4",
            label : "Action 4",
            url : "#action",
            items: [
                {
            		id : "action_4_1",
                    label : "Second Level Action 1",
                    url : "#action"
                },
                {
            		id : "action_4_2",
                    label : "Second Level Action 2",
                    url : "#action"
                },
                {
            		id : "action_4_3",
                    label : "Second Level Action 3",
                    url : "#action",
                    disabled: true
                }
            ]
        }
    ]
}

Localization of dropdown menus

In the sample data sets above you will notice the content for each menu items and button/link text. If you want all these to be localized, set the value of each labels to a textkey:


label : Sherpa.msg('textkey_label'),
