Overview

The Sherpa framework is highly configurable. Configurations are in place to adapt to the diverse needs in User eXperience (UX) prototype modeling. These configurations allow prototypes to run locally or from remote servers. In addition, different javascript libraries can be configured when the prototype page loads. This is done so the framework can be extended to fit your specific needs.

Out of the box, Sherpa is loaded with everything we use to build prototypes ourselves and very seldom you might need to add or remove things. There are several places to configure Sherpa show below.

Core Configuration

By editing assets/config/global_config.js you can change core settings. This javascript file contains the core of Sherpa such as paths, prototype default application, Javascript core libraries, CSS core files, default css and default locale.

Configuration Overrides

Each html file can contain override configurations. This allows you to create different pages that might use different libraries or helper files.

In the head of your HTML document you will notice a script tag. All of the override variables are commented out for your convenience:

<head>

	...

	<script type="text/javascript">
	    var SHERPA_CONFIG_OVERRIDES = {};
		SHERPA_CONFIG_OVERRIDES.CSS_LOCAL_APP = "assets/css/myCustomStyles.css";
		SHERPA_CONFIG_OVERRIDES.PROTOTYPE_APP = "myCustomApp.js";
		SHERPA_CONFIG_OVERRIDES.ENABLE_GRIDSET_OVERLAY = true;
		SHERPA_CONFIG_OVERRIDES.JS_CORE_LOAD = [
		    { myCustomJS: "assets/js/myCustomJSLibrary.js"}
		]
	</script>

	... 

</head>

Custom Configurations

The file assets/js/app.js is where your prototype application is. Here you can add custom configuration variables to the global data view model. These can be used to set up conditionals to display certain components or not.

All you have to do is add an element to the addToConfigs object. The sample below would be available in the view model (viewModel.config.sample_custom_config).

addToConfigs = {
	sample_custom_config : true
}

Localized Settings

The JSON file assets/config/settings_en.json is one of many files that are specific to a locale and contain global configurations for things such as date formats, currency conversion, etc. The file settings_en.json loads by default unless a different locale is specified on the url or core configuration.

The contents of this file get placed into the viewModel.config object along with custom configurations: 

{
     "default_order_code": "fncwl44br",
     "default_date_format": "mm/dd/yyyy",
     "currency": {
          "conversion_rate": 1,
          "format_decimal": {
               "decimals": 2,
               "decimal_separator": ".",
               "thousands_separator": ","
          },
          "format_nodecimal": {
               "decimals": 0,
               "decimal_separator": ".",
               "thousands_separator": ","
          },
          "symbol": "$",
          "symbol_position": "left"
     },
     "sample_custom_config": true
}
